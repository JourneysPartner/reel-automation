最終更新: 2026-05-23

## ゴール（ユーザー要望）
1. **既存カルーセル（6/1〜6/27, 13本）をまず活用**。ただし文字の改行をルール準拠に再チェックし、必要なものは修正。
2. 修正版は**ネット上の承認ページで確認・承認**（リールと同じ）。
3. **決められた日付（schedule.yaml）に公開**。
4. 既存が尽きたら **cron が新規生成 → 承認 → 公開**（全自動）。
5. 長期: 内容も API が考えて作成・公開まで自動。

## 現状
- 既存カルーセルは **ローカル `output/posts/`** のみ（リポジトリ未登録／gitignore）。各日付に `slide_1..8.png` + `slides.json` + `caption.md`。
- 改行は **Python `image_renderer.py` の旧ヒューリスティック**（`_find_natural_break`/`_prevent_orphan`）。リールの kuromoji 品質には未達。
- フィード統合のコードは実装済み（生成→GCS→承認(複数画像)→公開）。cron は「未生成の将来日付」を生成。
- **既存スライドには改行以外の保留課題も混在**（実例: 6/12 slide_2 = 比較が詰まって変な改行／アカウント名 `@guardian_tax`／利益500万）。文字は画像に焼き込み済み＝**修正には再レンダリングが必須**。

## 設計

### 改行ロジックの方針（リール品質に合わせる）
- 課題: リールは Node(kuromoji)、カルーセルは Python。
- **採用案: Python に純Python形態素解析「janome」を導入**し、reel の `jpWrap.js` と同等ロジックを `image_renderer` に実装:
  - 単語・複合名詞（消費税 等）を割らない／約物の直後・助詞の直前で折る／開き括弧の前で折る／表示幅ベース／孤立行回避。
  - 追加のシステム依存なし（辞書同梱）。CI も Python だけで完結。
- **カルーセル固有ルール**: 比較ラベル（個人：/法人：/ビフォー：/アフター：）は**各々を別行**に。番号リスト（①②）も改行（既存ロジックを踏襲・強化）。

### 既存カルーセル修正の範囲（要決定 → 下の確認事項①）
- (最小) 改行だけ直す。
- (推奨) この再レンダリングの機会に**保留中の修正キューも一緒に適用**（アカウント名 @guardian_tax_ac／6/12利益700万＋比較改行／6/3「逃すと、」改行／6/27「プライムデー後…」等）。公開前提なら少なくともアカウント名は要修正。

## フェーズ

### F1. 既存カルーセルの改行監査
- 13本×最大8枚のスライド本文を点検し、問題（単語分割・約物行頭・孤立行・比較詰まり）を一覧化。

### F2. カルーセル改行ロジックの改善（janome 導入）
- `image_renderer.py`（および本文整形 `_process_content_body` 等）に janome ベースの自然改行を実装。
- 比較ラベル別行・番号リスト改行ルールを強化。
- `requirements.txt` に janome 追加。

### F3. 既存カルーセルの再レンダリング
- `slides.json` を入力に、改善ロジックで `slide_*.png` を再生成（+ 確認事項①で決めた保留修正も適用）。
- 13本ぶん一括再レンダリングのスクリプトを用意。

### F4. 既存カルーセルを承認フローへ登録
- 各日付の `slide_*.png` + `caption.md` を GCS（`posts/<date>/`）へアップロード。
- スプレッドシートに行追加（type=carousel, status=review, preview=画像配列）。
- ChatWork に承認依頼通知 → **ネット上の承認ページで確認・承認**（リールと同一）。
- 一括登録スクリプトを用意（13本まとめて review 化）。

### F5. cron 調整（既存優先・尽きたら自動生成）
- 生成 cron: schedule.yaml の carousel 日付で、**スプレッドシートに既登録なら生成スキップ**（実装済の computeDue がそのまま機能）。
- 既存（〜6/27）は再生成されず、6/27 以降の新カルーセルだけ cron が自動生成。

### F6. 公開（実装済み）
- 公開 cron が承認済みカルーセルを予定日に投稿（カルーセルAPI対応済み）。

### F7. 長期フル自動
- 既存の generate/publish cron でリール・カルーセル両方を自動運用。
- 「内容も API が考える」: 現状は schedule.yaml が計画表。将来は**月次テーマ自動立案**（generate_month 相当を Sonnet で）も追加可能（オプション）。
